## 2. 核心能力

### 能力 1：智能代码补全

```
传统 IDE：
  你输入：d
  IDE 建议：def, delete, describe...（1-3 个选项）

GitHub Copilot：
  你输入：d
  Copilot 建议：一整行代码（可能对，可能不对）

Cursor：
  你输入：function.get
  Cursor 自动理解你要用 GET 请求获取数据
  建议完整的代码块（包括错误处理）
  还能理解你项目的 API 风格
```

### 能力 2：代码理解（关键创新）

```
Cursor 的"@"快捷键：

  当你在对话中输入 "@folder" 时：
  Cursor 自动分析整个文件夹的代码结构
    → 理解命名规范
    → 理解架构模式
    → 理解常用的库和框架

  结果：
    你："创建一个 API 端点"
    Cursor："我看到你用 Express.js, 路由在 /routes，需要输出 JSON"
    → 生成的代码自动符合你的项目风格

vs GitHub Copilot：
  Copilot 不理解项目整体
  看每行代码都是"孤立"的
  可能生成与项目风格不符的代码
```

### 能力 3：多行编辑和重构

```
场景：需要把 50 行代码的命名约定改掉

传统方式：
  1. Ctrl+H 打开"查找替换"
  2. 一个一个替换（可能有漏网的）
  3. 时间：10-20 分钟

Cursor 的方式：
  1. 选中代码块
  2. 告诉 Cursor："把所有 camelCase 改成 snake_case"
  3. Cursor 自动：
     - 识别所有变量名
     - 替换（包括调用处）
     - 更新测试
  4. 时间：1-2 分钟

加速比：10 倍
```

### 能力 4：自动生成测试

```
你写了一个函数：
  def calculate_tax(price, tax_rate):
    return price * (1 + tax_rate)

告诉 Cursor："生成单元测试"

Cursor 自动生成：
  import unittest

  class TestCalculateTax(unittest.TestCase):
    def test_normal_case(self):
      self.assertEqual(calculate_tax(100, 0.1), 110)

    def test_zero_rate(self):
      self.assertEqual(calculate_tax(100, 0), 100)

    def test_negative_price(self):
      # 边界情况测试
      ...

时间节省：80% (原来需要手写 30 分钟)
```

## 6. 使用建议

### 什么时候应该用 Cursor？

```
✅ 绝对应该：
  - 你是全职程序员
  - 你用 VS Code
  - 你想提升编码速度
  - 你公司允许（数据安全OK）

⚠️ 考虑中：
  - 你是兼职程序员
  - 你的项目很小
  - 成本 $20/月 比较紧张

❌ 可能不适合：
  - 你用 JetBrains IDE（改用 JetBrains AI）
  - 你是初学者（应该先学编程基础）
  - 你的项目涉及极端机密信息
```

### 最佳使用模式

```
场景 1：日常开发（重度使用）
  - 绑定快捷键（Cmd+K 打开 AI）
  - 经常用 @symbol 引用代码库
  - 用自动生成测试功能
  预期生产力提升：3-5 倍

场景 2：学习新技术（中度使用）
  - 有问题就问 Cursor
  - 自己修改生成的代码
  - 理解背后原理
  预期学习加速：2-3 倍

场景 3：Code Review（轻度使用）
  - 让 Cursor 审查代码（找 bug）
  - 自己做最终决策
  预期 bug 检出率提升：20-30%
```
